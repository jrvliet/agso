Staying organized:
  • Put a README files in each directory.
  • Use (a ridiculous amount of) physical and digital post-it notes.
  • Keep a research notebook. It helps you track your work over time, keep a record of all calibration/reduction/programming steps, serve as a reference place to track down that comment your adviser made a year ago that you wrote down and then forgot, and helps you work out your research ideas --- all in one little package. Your "notebook" can be a physical book, a folder with sheets of paper, a computer file, an HTML file --- the means doesn't matter as long as you are detailed and consistent.
  • Keep track of deadlines and events with Google Calendar.
Managing references:
  • Use the NASA/ADS Journal/Volume/Page Query Form to search using a reference as printed in a paper bibliography.
  • Use the NASA/ADS private library to keep track of your references.
  • The Mendeley Desktop helps you organize, tag, comment on, and search your references. You can synchronize your collection online and offline, allowing you to peruse articles without an internet connection.
  • The Harvard Bibtex style takes the irritation out of a bibliography, and this style formats your references as (Last name, Year). In your Latex file put the following two lines right before the \end{document} line:
    \bibliography{myrefs} \bibliographystyle{elsart-harv}
    At the bottom of every NASA/ADS listing is a link "Bibtex entry for this abstract", which gives you the Bibtek-formatted info for that particular listing. The myrefs.bib file can have entries for every paper you've ever read, and Bibtex will only list the references you use in your paper with \citet{} and \citep{} commands. AstroNat has useful Latex, Bibtex, and Perl software to allow the use of BibTeX to organize bibliographies when writing articles for Astronomy journals.
  • I always use the shortcut name in my Bibtek file when I mention a reference, even if it's only in a note to myself or an outline. That way I don't have to look through my bib file to check what nickname I used for the reference.
Handy computer tricks:
  • When I have to work in several directories at once, I open several shells in the terminal I'm working in. This way I can do something in one shell, press alt-arrow to get to another shell/directory and continue working. You could also open up several terminals but I like to use many tabs rather than many windows.
  • I write shell scripts to do things that I do constantly, like backing up files or making LaTex documents. The only two requirements are the shell line at the top (for example #/bin/cshrc!) and making the file executable with chmod. These are usually two- or three-liner codes like a series of search and replaces to get filenames (from a simple ls) in the right format to run, or a compile-and-scp-files command.
  • If I have to run a code many times for many different files, I create an executable file with the commands I want to run.
  • Using aliases in your shell file:
    Alias l, la and ll to ls, ls -a and ls -l, respectively.
    Alias specific version of python needed for matplotlib.
    Alias misspellings of common commands (like emas, emcas, emcs, etc.) to the properly-spelled command (emacs).
    Alias commonly-used scp addresses and directory paths.
UNIX one-liners:
  • du -h to check out hard disk space in convenient-to-read units
  • rsync -arvz --progress --delete path_backup_from path_backup_to is handy for backups
  • sort fileName | uniq -w number gets all the unique lines in a file based off of the first number of characters, useful if you have names of stars and lots of observations of them, and you want to know just how many stars there are in a directory
  • find ./ -name filename returns path info for string matching 'filename'. It takes wildcards also. It's like ls -R | grep filename but better.
  • head filename and tail filename is a quick way to view the beginning or end of a file, respectively.
  • ls -ld */ lists all subdirectories
  • sed -i 's/bluh1/bluh2/g' *.dat is a simple search and replace of all instances of bluh1 by bluh2 in files *.dat
  • rename .dat .txt ?mag.dat batch file renaming
